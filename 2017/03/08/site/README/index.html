After all of these, I want to build a site to show my blogs or anything what I try to realize. This post is used to show a piece of my mind about site’s framework.

Traditional or SPA?

According to baidu.com, SPA need to take care of SEO. But what’s SEO? With my dingding thing? I just write this to find a job.
I’ve tried to build these versions of project info (don’t care about the name):

  • V1.x (16)
    Blind to write. Even not used vue’s component, just tried to be familiar with vue syntax. It still has jQuery,Bootstrap(material-design-bootstrap). And at that time, Vue just as version 1.x
    What the build tools is? Webpack? Could I eat it?

  • V2.x (16)
    Updated vue version to v2, and something happen to vue2 & jQuery: Vue2 has occupied $ symbols, and I could not use then at same time by <link>. Then I started to use npm,node,webpack to build my project. By import jQ from 'jquery' to use both of them.
    Although I used components, always failed to run webpack-dev-server. I could only run npm run build & start nginx (yes, it’s windows). Every times I had any change would be suffering.

  • V3.x (16-17)
    I decided to reconstruct my web page to correct use webpack, and started to use Vue-Router, without jQ,Bootstrap etc.. and use muse-ui to take the place of them.
    In this process, components’ data stream cause I had a lot of props and this.$emit to write. So it was naturally introduced vuex’s state management.
    Then I didn’t write in a hurry after the introduction of the Vue-Resource to connect backend on LeanCloud, even without transition (Spicy chicken garbage website).

So in version 4, I would still use Vue’s family, and try to put improve css to be the most impornent thing of mine. Of course using webpack is, too.
Element would be my target to copy I could learn from Element’s UI components.

What would I use?

Of course I use Vue2 bucket family.
Might use Element’s UI components first, then write my own.
