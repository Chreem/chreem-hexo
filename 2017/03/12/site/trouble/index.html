vue runtime-only

  • Recurrence
    控制台虚区：index.bundle.js:1876 [Vue warn]: You are using the runtime-only build of Vue where the template compiler is not available. Either pre-compile the templates into render functions, or use the compiler-included build. (found in <Root>)

  • Reason
    import Vue from 'vue'默认打包/node_modules/vue/dist/vue.runtime.common.js，即runtime-only不带模板编译器，而目录结构中index.js使用了template属性，

    From Vuejs.org

    Runtime + Compiler vs. Runtime-only
    If you need to compile templates on the fly (e.g. passing a string to the template option, or mounting to an element using its in-DOM HTML as the template), you will need the compiler and thus the full build:

  • solve
    there’s two ways to solve this problem:

    1. 页面内容不多，可将index.js,index.html,index.css直接塞进index.vue中，用vue-loader解析
      继续使用runtime-only mode：

    2. runtime + compiler
      修改webpack配置，import默认引入改为vue.js而非vue.runtime.js

axios delete don’t have request.body

根据文档解释：

data is the data to be sent as the request body. Only applicable for request methods ‘PUT’, ‘POST’, and ‘PATCH’. When no transformRequest is set, must be of one of the following types:

  • string, plain object, ArrayBuffer, ArrayBufferView, URLSearchParams
  • Browser only: FormData, File, Blob
  • Node only: Stream

所以为了图懒-，- 必须指定数据放在data里，而且为了提供提交表单处理支持，添加头部信息

测试

mocha 超时

Mocha提供了两种异步测试方式，分别是done();和返回promise

而出现使用promise超时若不是后端问题十有八九就是把promise放到done=>{}里去了

捕获failing

Mock的failing通过promise的.catch的异常捕获，所以如果自己强行写了个.catch就会导致结果passing
